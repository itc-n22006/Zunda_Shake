DNSを略さずに言うと

Domain Name System

umaskとは

新規に作成されるファイルやディレクトリのアクセス権限を指定するコマンド

HTMLを略さずに言うと

HyperText Markup Language

アーカイブとは

専用の保存領域に安全にデータを保存すること

クエリ(Query)とは

ソフトウェアに対するデータの問い合わせや要求などを意味する

インデントとは

文章の行頭に空白を挿入して先頭の文字を右に押しやること

DNSサーバの役割は

IPアドレスとドメイン名の紐付けを管理したり、IPアドレスとドメイン名の変換をやってくれるコンピュータのこと

chmod と chown の違いは

chmodは誰が何を出来るかを定義し、chownはファイルの所有者を変更するために使用される

アーキテクチャ(architecture)とは

抽象的あるいは基本的な構造や設計、動作原理、実現方式などを表す

メモリとストレージの違いは

メモリはデータなどの一時的な保管場所であるが、ストレージはアプリケーションやデータを永久的に保管しておく場所

イーサネットとは

パソコンやゲーム機といったデバイスに、ケーブルを使ってインターネット接続する通信規格のひとつ

デーモンとは

UNIX系のOSなどにおける常駐プログラムの呼び名

SSLとは

2台のサーバー間で送信されるデータを暗号化することによりインターネット接続を保護する標準的なテクノロジー、Secure Sockets Layer

クラウドサーバとは

ネットワーク経由で使う形態になっているサーバ、あるいは仮想サーバのこと

SQLとは

データベース作成と操作のための標準言語、Structured Query Language

APIとはどんなものか

データなどを、他のプログラムから呼び出して利用するための手順やデータ形式などを定めた規約のこと、Application Programming Interface

KVMを略さずに言うと

Kernel-based Virtual Machine 仮想マシンを作成できるソフトウェア機能

スキーマ(schema)とは

何かの「構造」を指す用語

プロパティ(property)とは

対象の持つ設定や状態、属性のこと

Nullとは

なんのデータも含まれない状態のこと

Sambaとは

Windowsネットワークの諸機能をWindows以外のシステムで利用できるようにするソフトウェア

NFSを略さずに言うと

Network File System

サブネットマスクとは

IPアドレスのうち、ネットワークアドレスとホストアドレスを識別するための数値

パーミッション設定とは

「書き込み」「読み取り」「実行」の制限設定を行うこと

アーカイブとは

専用の保存領域に安全にデータを保存すること

ハイパーバイザとは

バーチャルマシンを実行するための仮想化技術のこと、仮想化ソフトとほぼ同義

BPOを略さずに言うと

Business Process Outsourcing(アウトソーシング)

Kernelとは

Linuxの中核部分、ハードウェアとソフトウェアの連携担当

BIOSを略さずに言うと

Basic Input Output System

RAMを略さずに言うと

Random Access Memory

CPUとはどんなもの

コンピュータの頭脳で、計算とデータ処理を担当(Central Processing Unit)

プロトコル(Protocol)とは

コンピュータやデバイス間の通信規則

PHPの正式名称は

Hypertext Preprocessor(Web系のプログラムを作るときによく使われるプログラム言語)

NATを略さずに言うと

Network Address Translation, 全世界で通用するIPアドレス(グローバルIPアドレス)と仲間内でのみ通用するIPアドレス(プライベートIPアドレス)を紐付けて変換する技術

ハッシュ値とは

入力に対応する適当な値

アパッチ(Apache)とは

世界的に使用される著名なWebサーバソフトの1つ

パラメータ(parameter)とは

システムに影響を与える、外から入ってくる値

FQDNとは

DNSなどのホスト名、ドメイン名などすべてを省略せずに指定した記述形式のこと、Fully Qualified Domain Name

ビルドとは

プログラムのソースコードをあれやこれやして実際のプログラムを作る作業のこと

デプロイとは

作ったプログラムをしかるべき場所においたり、あれやこれやの設定をしたりして、使える状態にすること
